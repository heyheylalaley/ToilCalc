# Исправление: редирект на localhost:3000

## Проблема
После успешной авторизации через Google OAuth, Supabase перенаправляет на `localhost:3000` вместо вашего реального домена.

## Причина
В Supabase Dashboard неправильно настроен **Site URL**.

## Решение

### Шаг 1: Откройте Supabase Dashboard

1. Перейдите на https://supabase.com/dashboard
2. Войдите в свой аккаунт (если нужно)
3. Выберите ваш проект (кликните на него в списке проектов)

### Шаг 2: Перейдите в настройки API

В левом боковом меню найдите и нажмите на:
- **Settings** (Настройки) - иконка с шестерёнкой ⚙️
- Затем выберите **API** из подменю

Или перейдите напрямую по ссылке:
```
https://supabase.com/dashboard/project/qwtwezxoodqfmdqpzkkl/settings/api
```

### Шаг 3: Найдите поле Site URL

На странице настроек API вы увидите несколько разделов:
- **Project URL** (это ваш API URL)
- **Project API keys** (ключи API)
- **Config** (Конфигурация) - вот здесь!

Прокрутите страницу вниз до раздела **Config** или **Configuration**.

В этом разделе вы найдете поле:
- **Site URL** (или **Site URL (for OAuth redirects)**)

### Шаг 4: Измените Site URL

Найдите поле **Site URL** и измените его на:

**Для GitHub Pages:**
```
https://heyheylalaley.github.io
```

**Для локальной разработки:**
```
http://localhost:8000
```
(или другой порт, который вы используете)

### Шаг 5: Сохраните изменения

1. После ввода URL нажмите кнопку **Save** (Сохранить) внизу страницы
2. Или если кнопки Save нет, изменения могут сохраняться автоматически - просто подождите несколько секунд
3. Вы должны увидеть уведомление об успешном сохранении

**Важно**: Если поле Site URL не видно или неактивно, возможно нужно:
- Прокрутить страницу вниз
- Проверить, что вы в правильном разделе (Settings → API)
- Обновить страницу (F5)

### Шаг 4: Проверьте

1. Откройте ваше приложение на правильном домене (не localhost:3000)
2. Попробуйте войти через Google снова
3. После авторизации должно перенаправить на правильный домен

## Временное решение (если нужно прямо сейчас)

Если у вас уже есть URL с токеном (как в примере), вы можете:

1. Скопировать весь URL с токеном
2. Заменить `localhost:3000` на ваш реальный домен:
   ```
   https://heyheylalaley.github.io/#access_token=...
   ```
3. Открыть этот URL в браузере

Но лучше исправить Site URL в Supabase, чтобы это работало автоматически.

## Важно

- **Site URL** должен совпадать с доменом, на котором размещено ваше приложение
- Для GitHub Pages это обычно: `https://YOUR_USERNAME.github.io`
- Не используйте `localhost:3000` если приложение размещено на GitHub Pages
