# Где найти Site URL в Supabase Dashboard

## Пошаговая инструкция с картинками

### Способ 1: Через меню (самый простой)

1. **Откройте Supabase Dashboard**
   - Перейдите на https://supabase.com/dashboard
   - Войдите в аккаунт

2. **Выберите проект**
   - В списке проектов найдите и кликните на ваш проект
   - Или перейдите напрямую: https://supabase.com/dashboard/project/qwtwezxoodqfmdqpzkkl

3. **Откройте Settings**
   - В левом боковом меню найдите иконку **⚙️ Settings** (Настройки)
   - Кликните на неё

4. **Выберите API**
   - В подменю Settings выберите **API**
   - Или перейдите напрямую: https://supabase.com/dashboard/project/qwtwezxoodqfmdqpzkkl/settings/api

5. **Найдите Site URL**
   - Прокрутите страницу вниз
   - Найдите раздел **Configuration** или **Config**
   - В этом разделе будет поле **Site URL**

### Способ 2: Прямая ссылка

Просто откройте эту ссылку (замените на ваш project ID если другой):
```
https://supabase.com/dashboard/project/qwtwezxoodqfmdqpzkkl/settings/api
```

Прокрутите вниз до раздела **Configuration** → **Site URL**

## Что делать, если не видите Site URL?

### Вариант 1: Прокрутите страницу вниз
Поле Site URL находится внизу страницы настроек API, после всех ключей.

### Вариант 2: Проверьте раздел Configuration
Site URL находится в разделе **Configuration** (Конфигурация), который может быть свернут. Разверните его, если видите стрелку.

### Вариант 3: Используйте поиск на странице
Нажмите Ctrl+F (или Cmd+F на Mac) и введите "Site URL" для поиска на странице.

### Вариант 4: Проверьте через Authentication → URL Configuration
Иногда Site URL можно найти в:
- **Authentication** → **URL Configuration**
- Или **Authentication** → **Settings** → **Site URL**

## Как должно выглядеть поле Site URL

Поле обычно выглядит так:
```
Site URL
https://heyheylalaley.github.io
[текстовое поле для ввода]
```

Или может быть в виде:
```
Site URL (for OAuth redirects)
[___________________________]
```

## Что ввести в Site URL

Для GitHub Pages:
```
https://heyheylalaley.github.io
```

**Важно:**
- ✅ С протоколом `https://`
- ✅ Без слеша в конце
- ✅ Без пути (только домен)

## После изменения

1. Нажмите **Save** (если есть кнопка)
2. Подождите 10-30 секунд
3. Попробуйте войти через Google снова
4. Теперь должно перенаправлять на правильный домен

## Если все еще не можете найти

Попробуйте:
1. Обновить страницу (F5)
2. Выйти и войти в Supabase снова
3. Проверить, что у вас есть права администратора проекта
4. Использовать другой браузер

## Альтернативный способ через SQL

Если не можете найти через интерфейс, можно изменить через SQL Editor:

1. Откройте **SQL Editor** в Supabase Dashboard
2. Выполните запрос (но это не рекомендуется, лучше через интерфейс)

Но лучше использовать интерфейс - поле точно есть в Settings → API → Configuration.
